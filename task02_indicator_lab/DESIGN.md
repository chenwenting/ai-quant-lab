## 1. 产品概述

### 1.1 产品定位

一款**纯前端、零后端依赖**的 A 股技术指标交互分析工具。用户选择股票数据文件，动态调整指标参数，实时重绘图表，无需编写代码即可探索不同参数对技术指标信号的影响。

### 1.2 目标用户

| 用户类型 | 特征 | 核心需求 |
|---------|------|---------|
| 量化初学者 | 了解指标概念，但不会调参 | 可视化理解参数对指标的影响 |
| 技术分析者 | 有经验，想快速验证参数 | 批量对比不同参数组合的信号差异 |
| 教学演示 | 培训/分享场景 | 动态演示参数变化带来的图表变化 |

### 1.3 核心价值

- 📂 **本地数据，隐私安全** — 所有数据在浏览器本地处理，不上传服务器
- 🎛️ **参数实时可调** — 滑块/输入框调整参数，图表即时重绘
- 📊 **四指标联动** — RSI / MACD / 布林带 / ATR 同屏展示
- 🚫 **零依赖部署** — 单HTML文件，双击即用

### 3.2 各模块详细功能

#### ② 左侧控制面板

**A. 数据文件选择器**
- 支持拖拽上传 `.csv` 文件
- 支持点击选择本地文件（`<input type="file">`）
- 自动检测 CSV 格式（支持 `trade_date,open,high,low,close,vol` 等格式）
- 显示当前加载的文件名和数据条数

**B. 全局控制**
- 🔘 图表主题切换（浅色/深色）
- 🔘 价格涨跌颜色切换（A股红涨绿跌 / 美股绿涨红跌）
- 🔘 是否显示交易信号标注（金叉/死叉/超买/超卖）

**C. 各指标参数卡片**（可折叠）

每个指标一个折叠卡片，展开后显示该指标的所有可调参数：

| 指标 | 可调参数 | 控件类型 | 默认值 | 范围 |
|------|---------|---------|--------|------|
| MACD | 快线周期(FAST) | 滑块+数字输入 | 12 | 5~30 |
| MACD | 慢线周期(SLOW) | 滑块+数字输入 | 26 | 10~50 |
| MACD | 信号周期(SIGNAL) | 滑块+数字输入 | 9 | 5~20 |
| MACD | 柱状体系数 | 滑块+数字输入 | 2 | 1~5 |
| BB | 移动平均周期 | 滑块+数字输入 | 20 | 5~50 |
| BB | 标准差倍数 | 滑块+数字输入 | 2.0 | 1.0~4.0 |
| RSI | 计算周期 | 滑块+数字输入 | 14 | 5~50 |
| RSI | 超买阈值 | 滑块+数字输入 | 70 | 60~90 |
| RSI | 超卖阈值 | 滑块+数字输入 | 30 | 10~40 |
| ATR | 计算周期 | 滑块+数字输入 | 14 | 5~50 |
| ATR | 显示类型 | 单选（真实波幅/ATR/ATR%） | ATR% | — |

**D. 操作按钮**
- 🔄 重置所有参数（恢复默认值）
- 📷 导出图表为 PNG
- 📋 复制当前参数配置（JSON）

#### ③ 中央图表区

- 使用 **Plotly.js** 绘制交互式图表（支持缩放、拖拽、悬停提示）
- 2×2 网格布局，每个子图对应一个指标
- 子图之间**时间轴联动**（缩放一个，其他同步）
- 支持隐藏/显示某个子图（每个子图标题栏有 👁 按钮）

#### ④ 底部状态栏

- 数据时间范围：`2025-07-01 ~ 2026-07-01（243条）`
- 信号统计：当前参数下 MACD 金叉 N 次 / 死叉 N 次；RSI 超买 N 次 / 超卖 N 次
- 参数快照按钮：点击可复制当前所有参数为 JSON

## 4. 数据层设计

### 4.1 输入数据格式

工具接受标准 A 股日线 CSV，字段如下（前4个字段必须，其余可选）：

```csv
trade_date,open,high,low,close,vol,amount,pre_close,pct_chg
20250701,116.00,118.50,115.20,117.80,1234567,1234567890,115.50,2.07
...
```

### 4.2 数据验证规则

| 规则 | 说明 |
|-----|------|
| 必须有 close 列 | 无收盘价无法计算任何指标 |
| 日期格式自动识别 | `YYYYMMDD` / `YYYY-MM-DD` 自动转换 |
| 数据按日期升序排列 | 自动排序，无需用户预处理 |
| 最少数据量检查 | 不同指标要求不同最小数据量（见下表） |

| 指标 | 最小数据量 |
|-----|----------|
| RSI(N) | N+1 条 |
| MACD(F,S) | S+max(F,SIGNAL) 条 |
| BB(N) | N 条 |
| ATR(N) | N+1 条 |

### 4.3 内部数据结构

```javascript

}
```

## 5. 指标参数与计算规范

### 5.1 MACD（指数平滑异同移动平均线）

**参数**：
- `FAST`：快线 EMA 周期，默认 12
- `SLOW`：慢线 EMA 周期，默认 26
- `SIGNAL`：信号线 EMA 周期，默认 9
- `multiplier`：柱状图放大系数（A股惯例 ×2），默认 2

**计算公式**：
```
DIF = EMA(close, FAST) - EMA(close, SLOW)
DEA = EMA(DIF, SIGNAL)
MACD_hist = multiplier × (DIF - DEA)
```

**图表规范**：
- 上方子图：收盘价折线 + DIF/DEA 双线
- 下方子图：MACD 柱状图（红涨绿跌）
- 零轴用虚线标注
- 金叉（DIF上穿DEA）：▲ 绿色标记；死叉（DIF下穿DEA）：▼ 红色标记

### 5.2 布林带 BB（Bollinger Bands）

**参数**：
- `N`：移动平均周期，默认 20
- `K`：标准差倍数，默认 2.0

**计算公式**：
```
middle = SMA(close, N)
std = 标准差(close, N)
upper = middle + K × std
lower = middle - K × std
```

**图表规范**：
- 收盘价蜡烛图（红涨绿跌）
- 上轨（upper）：红色虚线
- 中轨（middle）：蓝色实线
- 下轨（lower）：绿色虚线
- 上下轨之间：半透明填充区域

### 5.3 RSI（相对强弱指标）

**参数**：
- `N`：计算周期，默认 14（Wilder 平滑）
- `overbought`：超买阈值，默认 70
- `oversold`：超卖阈值，默认 30

**计算公式**（Wilder 平滑）：
```
delta = close[i] - close[i-1]
gain = max(delta, 0)
loss = max(-delta, 0)
avg_gain = EMA(gain, N)   // Wilder 平滑
avg_loss = EMA(loss, N)
RS = avg_gain / avg_loss
RSI = 100 - 100 / (1 + RS)
```

**图表规范**：
- RSI 折线（紫色）
- 超买线（70）：红色虚线 + 上方红色半透明区域
- 超卖线（30）：绿色虚线 + 下方绿色半透明区域
- 超买区域标注「超买」；超卖区域标注「超卖」

### 5.4 ATR（平均真实波幅）

**参数**：
- `N`：计算周期，默认 14
- `displayType`：显示类型，枚举：`TR` / `ATR` / `ATR%`

**计算公式**：
```
TR = max(high - low, abs(high - prev_close), abs(low - prev_close))
ATR = SMA(TR, N)          // 或 EMA(TR, N)，默认 SMA
ATR_pct = ATR / close × 100%
```

**图表规范**：
- 主图：ATR 或 ATR% 折线（橙色）
- 辅助Y轴（右侧）：显示 ATR/close 百分比
- 波动率放大区域（ATR% > 均值+1STD）：黄色高亮背景

## 6. 交互设计

### 6.1 参数调整交互

```
用户操作                系统响应
─────────────────────────────────────────────────────
拖动滑块               → 图表 300ms 防抖重绘
                          （防止频繁计算导致卡顿）

修改数字输入框         → 失去焦点或回车后触发重绘
                          输入框实时验证，超出范围自动纠正

点击「重置」按钮       → 所有参数恢复默认值
                          弹出确认对话框：「确认重置所有参数？」

切换指标显示/隐藏      → 对应子图立即显示/隐藏
                          布局自动调整（隐藏时 2×1 或 1×2）
```

### 6.2 文件上传交互

```
用户操作                系统响应
─────────────────────────────────────────────────────
拖拽 CSV 文件到页面    → 页面出现拖拽高亮边框
                          释放后自动解析并加载

点击「选择文件」       → 打开系统文件选择器
                          只显示 .csv 文件

文件格式错误           → 弹出错误提示：
                          「文件格式错误：缺少 close 列，
                          请检查 CSV 格式」
                          同时给出标准格式示例
```

### 6.3 图表交互

- **悬停提示**：鼠标悬停显示该日期所有指标数值
- **框选缩放**：在任一子图上框选，所有子图同步缩放时间轴
- **重置缩放**：双击图表恢复原始视图
- **信号标注交互**：点击信号标记（▲▼），弹出该信号的详细信息（日期、价格、后续N日收益）

## 7. 技术架构

### 7.1 技术选型

| 层级 | 技术 | 理由 |
|-----|------|------|
| 图表引擎 | **Plotly.js** | 交互性强，支持子图联动，导出PNG方便 |
| 指标计算 | **原生 JavaScript** | 轻量，无依赖，计算速度足够 |
| 数据处理 | **Papa Parse** | CSV 解析库，处理大文件（>1万行）性能好 |
| UI 组件 | **原生 HTML + CSS** | 无需框架，保持单文件可移植性 |
| 样式方案 | CSS Variables | 支持主题切换 |

### 7.2 文件结构（单HTML方案）

```
indicator_tool.html          ← 唯一交付文件（所有代码内联）
├── <style>                 ← CSS 样式（含 CSS Variables 主题）
├── <div class="sidebar">   ← 左侧控制面板
├── <div class="chart-area">← 中央图表区
├── <div class="status-bar">← 底部状态栏
└── <script>                ← JavaScript 逻辑
    ├── CSVParser            ← Papa Parse 调用
    ├── IndicatorCalculator  ← 四个指标的计算函数
    ├── ChartRenderer        ← Plotly.js 图表渲染
    ├── ParameterController  ← 参数滑块/输入框绑定
    └── EventHandler         ← 用户交互事件处理
```

### 7.3 性能考虑

- 数据量上限：建议 ≤ 5000 条（日线约20年），超出时提示「数据量较大，部分交互可能变慢」
- 防抖策略：参数调整使用 300ms 防抖，避免频繁重绘
- 计算优化：指标计算函数使用 typed array，避免频繁对象创建

## 10. 附录

### 10.1 竞品参考

| 产品 | 优点 | 本工具的差异化 |
|-----|------|--------------|
| TradingView | 功能强大，指标丰富 | 本地数据，隐私安全；参数调节更直观 |
| 同花顺PC端 | 数据全面，免费 | 无需安装，单文件即开即用 |

### 10.2 风险与限制

| 风险 | 影响 | 缓解措施 |
|-----|------|---------|
| 大文件（>1万行）性能问题 | 交互卡顿 | 限制最大数据量；使用 Web Worker 计算 |

| 浏览器兼容性 | 旧版浏览器无法使用 | 要求 Chrome/Edge 90+ |
